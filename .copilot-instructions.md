## Project overview

Cluckbait is a UK chicken burger review platform built with Rails 8.1, Ruby 3.4, and SQLite3. It features user reviews, social features (friends, messaging, wishlists), Google OAuth sign-in, and 58-language i18n support.

## Tech stack

- **Backend:** Rails 8.1.2, Ruby 3.4.1, SQLite3 (multi-database: primary, cache, queue, cable)
- **Frontend:** Hotwire (Turbo + Stimulus), Tailwind CSS, Propshaft, Importmap (no JS build step)
- **Auth:** Devise with Google OAuth2 (OmniAuth)
- **Jobs:** Solid Queue (runs inside Puma, no separate worker)
- **Deployment:** Kamal + Docker, Thruster reverse proxy

## Code style

- Follow GitHub's Ruby style guide via `rubocop-github`
- Max line length: 118 characters
- Frozen string literals: disabled
- Run linter: `bundle exec rubocop`
- Run security scan: `bundle exec brakeman`

## Testing

- Framework: Minitest with FactoryBot (no fixtures)
- Run tests: `bundle exec rails test`
- Test structure: `test/models/`, `test/controllers/` (integration style), `test/integration/`
- Factories in `test/factories/` ŌĆö use `build(:model)` and `create(:model)`
- Devise helpers available in integration tests via `Devise::Test::IntegrationHelpers`
- Tests run in parallel with `parallelize(workers: :number_of_processors)`
- Background jobs tested with `ActiveJob::TestHelper` and `perform_enqueued_jobs` blocks

## Key conventions

- Controllers use standard REST actions; Devise controllers are customised under `app/controllers/users/`
- Models use counter caches on `reviews_count` for `users` and `chicken_shops` ŌĆö don't define `reviews_count` methods
- Background work (notifications, activities) uses `perform_later` jobs, not inline
- Security: `MessagesController` uses a `SHAREABLE_CLASSES` hash for polymorphic resolution (never `.constantize`)
- File uploads validate content type and size (avatars: PNG/JPEG/GIF/WebP, max 5MB)
- All routes are wrapped in `scope "(:locale)"` for i18n ŌĆö use `root_path` etc. without explicit locale in code

## i18n

- 58 locales in `config/locales/`
- English (`en.yml`) is the source of truth
- When adding user-facing strings, always use `t("key")` and add the English translation to `en.yml`
- Locale precedence: URL param ŌåÆ user preference ŌåÆ session ŌåÆ default (en)

## Environment variables

- Google OAuth: `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`
- Devise mailer: `DEVISE_MAILER_SENDER`
- Use `dotenv-rails` locally ŌĆö copy `.env.example` to `.env`
